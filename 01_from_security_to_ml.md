# From Breaking Systems to Building Them

*June 2026*

---

I've spent my career as a penetration tester — finding the cracks in systems before the wrong people do. It's taught me to think in internals, to trace execution paths, to ask why something works the way it does rather than just accepting that it does.

Now I'm pointing that same mindset at machine learning.

This isn't a pivot driven by hype. It's a deliberate decision to move toward the kind of engineering that I think matters most right now. The goal is specific: an AI/ML engineer role at Anthropic.

---

## Where I'm Starting From

I'm not starting from zero, but I'm not far from it either.

I have some Python experience — enough to read code and write basic scripts. I don't have a formal background in higher-level mathematics, linear algebra, or calculus. I haven't done a CS degree. What I do have is a strong instinct for how systems work under the hood, and a habit of not moving on until I actually understand something.

That's the foundation I'm building on.

---

## The Approach

I'm following a structured 17-week training plan covering Python fundamentals, data structures, system design, and AI safety — with GPT-2 internals study running in parallel the entire time.

Alongside that I'm working through research papers, technical books, and daily LeetCode problems using a learning-first workflow — no looking up answers, no skipping the reasoning, one concept at a time.

Every solution in this repo comes with a plain-English explanation of what the code is actually doing and why. Because if I can't explain it simply, I don't understand it yet.

---

## Why Document It

Partly accountability. Partly because the journey from security to ML engineering in public is more useful to someone else than a polished portfolio would be.

Every commit here is a real session. Every problem log shows the actual mistakes, not just the final answer.

That's the point.
